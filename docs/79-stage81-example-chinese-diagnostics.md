# 阶段 81：示例中文诊断复核

## 1. 本阶段目标

阶段 81 从用户集成视角复核示例输出，优先清理会直接出现在用户终端里的英文提示。

本阶段不新增 API、不改变协议行为、不修改签名逻辑，也不触碰 Crypto Gate 或破坏性 lab 计数。

## 2. 发现的问题

阶段 79 审计发现 `ReactiveMinioLiveExample` 中仍有英文终端提示，覆盖缺少配置、bucket 创建流程、S3 错误和对象响应头等场景。

这些提示不影响 SDK 功能，但会让中文用户在第一次运行示例时难以快速定位问题，也不符合“新增注释和文档中文化”的项目约束延伸目标。

## 3. 本阶段修正

`ReactiveMinioLiveExample` 的终端输出已统一改为中文：

- 缺少配置时提示应在 `minio-local.properties` 或环境变量中设置哪个键。
- bucket 创建、对象上传/下载/删除、响应头打印改为中文。
- S3 错误输出改为中文状态码和响应体说明。
- 默认示例内容改为中文文本。

## 4. 保留边界

- 示例中的类名、方法名、变量名仍保持 Java/SDK 命名风格，不强行翻译。
- 测试方法名仍保持既有英文命名，避免无意义的大规模改名。
- HTTP header、bucket、object、KMS、STS、raw 等技术名词保留原词。

## 5. 验证口径

本阶段验证应覆盖：

- 示例代码可编译。
- JDK8/JDK17 测试与 live 仍通过。
- JDK21/JDK25 `test-compile` 仍通过。
- 变更文件不包含真实凭证。
- 双分支示例文件同步。
